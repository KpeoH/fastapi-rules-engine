# Движок правил рекламных кампаний

Сервис на FastAPI для автоматического управления статусами рекламных кампаний на основе бизнес-правил.

## Архитектура движка правил
Движок спроектирован с упором на расширяемость и принцип Open/Closed.
* **Изолированность:** Все бизнес-правила вынесены в отдельные классы (`BudgetRule`, `StockRule`, `ScheduleRule`), реализующие единый интерфейс `Rule`.
* **Оркестрация:** Оркестратор (`CampaignEvaluator`) не содержит бизнес-логики, а лишь последовательно прогоняет кампанию через конвейер проверок.
* **Тестируемость:** Правила изолированы от БД и фреймворка, что позволяет покрыть их unit-тестами без внешних зависимостей.


## Как запустить проект

1. Убедитесь, что у вас установлен Docker и Docker Compose.
2. Запустите сборку и старт контейнеров:
   ```bash
   docker-compose up --build
   ```
Сервер запустится и автоматически накатит миграции на базу данных. Swagger UI: http://127.0.0.1:8000/docs

## Как запустить тесты

Тесты проверяют изолированную бизнес-логику (граничные случаи бюджета, расписания, остатков и приоритетов) без необходимости поднимать БД.  

```Bash
cd test
pytest -v
```
